[Micro] Intro sobre Blockchain

¿Qué es un Smart Contract?

Un Smart Contract es, básicamente, un programa que tiene un set de instrucciones predefinidos y que funciona sobre una plataforma Blockchain.


Existen diferentes tipos pero nos enfocaremos en plataforma Ethereum

¿Cómo funciona?

  • OPCodes + complejos (que Bitcoin)
  • EVM based blockchains
  • Consumen unidades de gas GAS
  • Los ejecutan los nodos/miners que participan en la red EVM
  • Aplicaciones descentralizadas / transparentes (DApps)

Existen diferentes tipos:

  1. EVMs (Ethereum)
  2. Chaincode (Hyperledger)
  3. Otros

Componentes de un Smart Contract

  • ABI (Application Binary Interface)
  • Source Code
  • Bytecode

Algunas particularidas de los Smart Contracts

  1. Las direcciones son aleatorias
  2. Las direcciones públicas NO poseen privkeys asociadas a la misma
  3. Los contratos pueden contener balances (Ether/Tokens)
  4. Una vez deployeados los contratos no pueden modificarse
  5. Se pueden decir que existen dos tipos de llamadas a contratos (Calls y Transactions)

Configurando el entorno

Para testear nuestros Smart Contracts necesitamos una EVM disponible donde poder probar la funcionalidad sin costo asociado de GAS/Ether real


    Componentes:

  • Compilador
  • EVM
  • IDE
  • Framework de testing
  • Conexión RPC

Herramientas a utilizar

  1. Solc (compiler)
  2. EVM (TestRPC / Ganache)
  3. Sublime/Atom/VSCode/otros <- (Ninguno es el mejor no quiero desatar una guerra nerd)
  4. Truffle (unit testing)
  5. Nodo propio o RPC API / Web3 (localhost o Infura / QuikNode)
